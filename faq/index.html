Frequently asked questions regarding the PERCS ecosystem.

1. Security & Moderation
Use the .antinuke (or .an) command to initialize our comprehensive protection system. This monitors mass deletions and unauthorized administrative changes in real-time.
The /lookup command traces user identity signatures to detect potential alt accounts. For general data, use .whois or .ui.
While .ban removes a user from the server, .hardban permanently firewalls the user from the server to prevent reentry through any means.
Enable .antilink for advanced filtering with domain whitelisting and punishment escalation protocols.
2. Utility & Content

Admins can use /welcome set to configure join messages and /welcome test to verify the appearance. Similar controls exist for /leave messages.
Use /add-level-role to assign a specific role reward to a target level. You can view all active rewards using /level-roles-list.
3. Global Services
